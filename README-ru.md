# CertManager (PKI Web Interface)

CertManager — это легкое Docker-приложение для управления собственной инфраструктурой открытых ключей (PKI). Оно позволяет легко создавать, управлять и распространять центры сертификации (CA), серверные и клиентские сертификаты (для mTLS) через современный интерфейс или напрямую через API.

## 🚀 Основные возможности

- **Управление центрами сертификации**: Создание самоподписанных корневых CA или промежуточных CA. Импорт внешних CA.
- **Генерация сертификатов**: Выпуск серверных TLS/SSL сертификатов и клиентских сертификатов с настраиваемыми альтернативными именами субъектов (SAN) и сроками действия.
- **Жизненный цикл сертификатов**: Просмотр деталей (включая издателя), отзыв или безвозвратное удаление сертификатов.
- **Варианты экспорта**: Скачивание сертификатов в форматах `.crt`, `.key` или `.p12` (PKCS#12).
- **Безопасность**:
  - Веб-интерфейс защищен JWT (JSON Web Tokens).
  - API эндпоинты защищены API-ключом.
  - Docker-контейнеры работают от имени безопасного пользователя без прав root (`node`).
- **Современный интерфейс**: Фронтенд на React с пагинацией, фильтрацией и адаптивным дизайном.

## 🛠 Технологический стек

- **Frontend**: React, TypeScript, Vite, Nginx.
- **Backend**: Node.js, Express, TypeScript, `node-forge` (для PKI/криптографии).
- **Развертывание**: Docker и Docker Compose.

## 🏃 Быстрый старт

### Требования
- [Docker](https://docs.docker.com/get-docker/)
- [Docker Compose](https://docs.docker.com/compose/install/)

### 1. Клонируйте репозиторий
```bash
git clone <url-вашего-репозитория>
cd cert-man
```

### 2. Настройка окружения (опционально)
Вы можете отредактировать файл `docker-compose.yml`, чтобы изменить пароли и секреты по умолчанию:

| Переменная | Значение по умолчанию | Описание |
|------------|-----------------------|----------|
| `ADMIN_PASSWORD` | `admin` | Пароль для доступа к веб-интерфейсу. |
| `JWT_SECRET` | `supersecret_jwt_key` | Секрет для подписи сессионных токенов. |
| `API_KEY` | `my-secret-api-key` | Ключ для прямого доступа к API бэкенда. |
| `BACKEND_URI` | `http://backend:3001` | Внутренняя DNS-маршрутизация Docker для Nginx. |

### 3. Сборка и запуск
Запустите приложение в фоновом режиме:
```bash
docker compose up --build -d
```

### 4. Доступ к интерфейсу
Откройте браузер и перейдите по адресу:
**http://localhost:3000**

Войдите, используя `ADMIN_PASSWORD` (по умолчанию `admin`).

## 📖 Руководство по использованию

1. **Создание корневого CA**: Нажмите `+ New CA`, оставьте поле "Parent CA" пустым и заполните детали.
2. **Создание промежуточного CA**: Нажмите `+ New CA`, выберите существующий корневой CA в выпадающем списке "Parent CA".
3. **Выпуск сертификата**: Нажмите `+ New Certificate`, выберите CA, который его подпишет, выберите тип "Server" или "Client" и укажите Common Name (CN) и SAN (например, `example.com, 192.168.1.5`).
4. **Скачивание**: Нажмите кнопки `CRT`, `KEY` или `P12` рядом с сертификатом, чтобы скачать сгенерированные файлы.

## 💾 Хранение данных

Все сгенерированные сертификаты, приватные ключи и база данных в формате JSON хранятся в Docker Volume с именем `cert-data`.
- Путь внутри контейнера: `/app/data`
- Для резервного копирования сертификатов вы можете сделать копию содержимого этого тома Docker.

Если вы хотите примонтировать локальную папку вместо тома Docker, измените `docker-compose.yml`:
```yaml
    volumes:
      - ./my-local-certs-folder:/app/data
```

## 🛡️ Заметки по безопасности

- Docker-контейнер бэкенда использует многоэтапную сборку и работает от имени непривилегированного пользователя `node` для минимизации рисков безопасности.
- **Продуктовое использование**: При выводе сервиса в интернет настоятельно рекомендуется разместить его за обратным прокси-сервером (например, Traefik или Nginx Proxy Manager) для обеспечения работы интерфейса по HTTPS.
- Приватные ключи хранятся в открытом виде внутри тома Docker (`/app/data/keys`). Убедитесь, что файловая система вашего хоста защищена.
